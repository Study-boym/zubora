# ユニット生成（実装単位の分解）

Phase 1 の実装を細かい作業単位に分解したものです。

## Phase 1 実装ユニット一覧

| ユニットID | 名称 | 内容 | 依存ユニット |
|---|---|---|---|
| U-01 | Cognito認証基盤 | ユーザー登録・ログイン・JWT発行 | なし |
| U-02 | ユーザー設定API | 好み・アレルギー・通知時刻をDynamoDBに保存するCRUD API | U-01 |
| U-03 | Bedrockチャット連携 | Claude Sonnetへのリクエスト送受信ラッパー（バーマスタープロンプト付き） | なし |
| U-04 | WebSocketチャットAPI | API Gateway WebSocket + Lambda によるリアルタイムチャット処理 | U-01, U-03 |
| U-05 | 会話履歴保存 | チャットのやり取りをDynamoDBへ保存・取得するモジュール | U-01 |
| U-06 | レコメンド生成Lambda | Bedrock Haikuを呼び出して献立・服装案のテキストを生成 | U-02 |
| U-07 | EventBridge定時トリガー | 朝6:30・7:00・8:00・17:00 に U-06 を起動するスケジューラ設定 | U-06 |
| U-08 | SNSプッシュ通知送信 | 生成したレコメンドをPinpointでデバイスへ配信 | U-07 |
| U-09 | フロントエンド：チャット画面 | WebSocket接続・メッセージ送受信・履歴表示 | U-04, U-05 |
| U-10 | フロントエンド：設定画面 | 通知時刻・好み・アレルギーの登録フォーム | U-02 |
| U-11 | フロントエンド：レコメンド表示 | 最新レコメンドをホーム画面に一覧表示 | U-06 |

## 依存関係図

```
U-01 (認証)
  └── U-02 (設定API)
        └── U-06 (レコメンド生成)
              ├── U-07 (定時トリガー)
              │     └── U-08 (プッシュ通知)
              └── U-11 (レコメンド表示画面)
        └── U-10 (設定画面)
  └── U-04 (WebSocketチャットAPI)
        ├── U-03 (Bedrock連携) ←─ 独立
        └── U-09 (チャット画面)
              └── U-05 (会話履歴)
```
