# AIDLC 人をだめにする案

## 背景
AIDLC ハッカソンにて、人×AIの力を活用して『人をだめにするサービス』作成を考慮する大会があります。
世の中の情勢や、人が一体何を求めどんなサービスを作成することでダメにすることができるか考慮しています。

## ゴール
考慮した案を以下のフェーズまで考慮すること
https://github.com/awslabs/aidlc-workflows/blob/main/aidlc-rules/aws-aidlc-rules/core-workflow.md
```
開始段階
目的：計画策定、要件収集、およびアーキテクチャ上の意思決定

焦点：何を作るのか、そしてなぜ作るのかを明確にする

開始段階の段階：

・ワークスペース検出（常時）
プロジェクトが新規か既存なのかをAIが自動で判別するフェーズ

・リバースエンジニアリング（条件付き - 既存設備のみ）
既存のコードがあれば解析するフェーズ


・要件分析（常に適応的な深さ）
何を作成するのかをAIと深堀しながら分析・整理するフェーズ

・ユーザーストーリー（条件付き）
ユーザー視点で機能についてを定義するフェーズ

・ワークフロー計画（常に）
要件分析をもとに、実際にどのように開発を進めていくかの計画を整備・決定するフェーズ

・アプリケーション設計（条件付き）
本格的なアーキテクチャ設計(構成図)が必要なときに設計するフェーズ

・ユニット生成（条件付き）
作業を細かい実装単位に分解する必要があるときに生成するフェーズ
```

## 依存に対する考慮点
サービス案を出すにあたり、依存観点について洗い出します。

### 人は日常的に何に依存して過ごしているのか？
- 行動・習慣系
    - スマートフォン
    - SNS(X, Instagram)
    - 動画媒体(Netflix, Youtube, Amazon Prime)
    - ショッピング
- 物質系
    - カフェイン（コーヒー、エナジードリンク）— 摂取しないと頭痛や倦怠感が出やすい
    - アルコール — 習慣化しやすく、耐性がつきやすい
    - 糖分・甘いもの — 血糖値の変動で「また食べたい」感覚が生まれる
- 心理・関係系
    - 承認欲求（「いいね」を求める行動）
    - 特定の人間関係への依存（共依存）
    - 過食・制限食など食にまつわる行動パターン
- 環境・感覚系
    - 音楽・ノイズ（無音が落ち着かない）
    - 特定の場所・空間（カフェでないと作業できない）

### なぜ人間は依存してしまうのか
- ストレス
    - ストレス解消きっかけでやめられなくなった
- 退屈・刺激不足
    - 空虚な状態を埋めるための行動。スマホを開く多くのケースはこれに近い。
- アイデンティティ・習慣の固定化
    - 自分はコーヒーを飲む人間だ」という自己イメージが行動を固定する。やめる理由がないまま継続される。
-  社会的同調圧力
    - 飲み会のアルコール、職場のコーヒー文化など、集団に属するための行動が習慣化するケース。
-  生理的快楽・報酬回路
    - ストレスとは無関係に、単純に「気持ちいいから」。糖分やカフェインは脳の報酬系を直接刺激する。
- 時間構造の代替
    - 仕事が終わった後にNetflixを見るなど、「次に何をするか」の意思決定コストを減らすルーティン化。
- 不確実性の回避
    - 予測可能な刺激（同じ作品の繰り返し視聴、同じ店に通うなど）で安心感を得る。新奇なものより既知のものを選ぶ。

上記のように複数ある依存の原因の中で、スマホは退屈・報酬回路・習慣化を同時に刺激できる唯一のデバイスであるため、依存性と替えがきかない存在である

### 普段の生活で必ず必要なこと
以下は日常で過ごすうえで必ず必要な項目だが、外に出なくてもスマホなどで対応できてしまっている
- 睡眠
- 食事
- 買い物

| カテゴリ | 時間 | 割合 | データ種別 |
|---|---|---|---|
| 睡眠 | 7h 54m | 33% | 実測・信頼度高 |
| スマートフォン・PC | 3h 02m | 13% | 実測・信頼度高 |
| テレビ（20〜40代平均） | 1h 33m | 6% | 調査値・参考 |
| 仕事・学校 | 6h 04m | 25% | 調査値・参考 |

> 出典：総務省統計局「社会生活基本調査（令和3年）」睡眠7h54m、総務省情報通信政策研究所「情報通信メディアの利用時間と情報行動に関する調査（令和6年度）」インターネット181.8分、令和5年度調査より20代54分・30代95分・40代130分の平均をテレビとして算出

### スマートフォン・PC 3h02m の内訳（テレビ除く）

| 用途 | 時間/日 | 割合 |
|---|---|---|
| 動画配信（YouTube・Netflix等） | 55分 | 30% |
| SNS・メッセージ | 53分 | 29% |
| ウェブ閲覧 | 35分 | 19% |
| 仕事・学習（オンライン） | 25分 | 14% |
| ゲーム・その他 | 14分 | 8% |

> 出典：総務省情報通信政策研究所（令和6年度）インターネット利用時間181.8分をベース、SNSはDataReportal 2024 Japan（53分/日）を採用

### データから読み取れる示唆

- **スマートフォン・PCは1日の約13%（3時間2分）**、テレビ（20〜40代）は約6%（1時間33分）— 合計デジタル・映像消費は約19%
- 全年代のテレビ平均は154.7分だが、**50代未満では93分程度まで低下**。ターゲットはスマホ中心の生活圏
- スマートフォン利用のうち **動画配信・SNSで約59%** を占め、受動的コンテンツ消費が中心
- SNS利用はDataReportalによる実測値53分（日本は世界最低水準）。メッセージ込みでもスマホ時間の約3割
- 睡眠・食事・仕事を除いた「自由に使える時間」はごくわずかであり、その大半をスマホが占有している

→ 50代未満ではテレビよりスマホが「空き時間の受け皿」として機能しており、スマホ動線上のサービス設計が依存を誘導しやすい

## 現代人が抱える悩みと意思決定負荷
 
依存の構造を踏まえ、現代人が日常的に抱える「悩みや負荷」を整理する。これがサービスが解決すべき課題の出発点となる。
 
### 現代人の主な悩み領域
 
| 領域 | 代表的な悩み | 特徴 |
|---|---|---|
| 決断疲れ | 選択肢が多すぎて選べない・選ぶこと自体が疲れる | 毎日繰り返し発生する |
| 慢性的ストレス・こころの不調 | 健康リスクとして「ストレス」を挙げる割合が20年で3倍に増加（厚労省2024年白書） | 30〜40代が特に深刻 |
| 疲労の蓄積 | 「疲れている人」が推計7,162万人（計測以来のピーク）、睡眠5時間未満が増加 | 睡眠不足と悪循環 |
| 失敗恐怖・行動抑制 | SNSで他者の失敗が可視化され、「安全な選択」しかできなくなる | 特に若い世代に顕著 |
| きずな依存・孤独感 | SNSの人間関係は負担だが、抜け出せない。依存者の52%が「SNS上の人間関係に負担を感じている」 | 日本人特有の傾向 |
 
### 意思決定負荷の深刻さ
 
人は1日に約3.5万回の意思決定をしていると言われている。現代では選択肢の爆発的な増加により、この負荷がさらに増大している。
 
**決断疲れ（Decision Fatigue）** とは、意思決定を繰り返すことで脳が疲弊し、後半になるほど「どうでもいい」と投げやりな選択をしてしまう現象。Netflixで何を見るか決められず結局閉じる、というのがその典型例。
 
#### 意思決定が発生する主な場面
 
意思決定の場面は大きく4領域に分けられるが、「依存しやすいサービス」を設計するうえでは、**繰り返し発生する日常的な意思決定**が最も有効なターゲットになる。
 
| 領域 | 代表的な場面 | 依存しやすさ |
|---|---|---|
| **日常生活** | 献立・食事・買い物・服装・余暇の使い方 | ◎ 毎日繰り返される |
| 仕事・業務 | メール返信・タスク優先順位・会議での発言 | ○ 毎日発生するが職種依存 |
| 消費・購買 | EC商品選択・サブスクプラン・旅行先 | ○ 頻度は高いが散発的 |
| キャリア・人生 | 転職・住まい・学習テーマ | △ 重大だが頻度が低い |
 
→ **日常生活領域**は毎日必ず発生し、失敗しても取り返しがつく軽さがある。そのため「AIに任せる」習慣が最もつきやすく、依存ループが形成されやすい。
 

### 孤独感・つながり希薄化の深刻さ

現代人はSNSによって表面上「常につながっている」状態にあるが、その実態は孤独感の拡大と逆相関している。内閣府「孤独・孤立の実態把握に関する全国調査（2023年）」では、孤独を「しばしば・常に感じる」と答えた人が全体の約40%に上り、特に20〜40代で増加傾向にある。

**デジタル孤独（Digital Loneliness）** とは、オンラインでの接触が増えるほど対面でのつながりが減少し、承認を「いいね」の数で測るようになることで、本質的な安心感が得られなくなる状態。SNSを見るほど他者との比較が生まれ、自己肯定感が下がるという悪循環も報告されている。

また、AIへの相談という新たな行動パターンも孤独の構造に影響を与えている。恋愛・人間関係・メンタルの悩みをAIに打ち明けるユーザーが増加しており、その背景には「人間に話すと気を遣わせる」「判断されたくない」「いつでも聞いてもらえる」といった**開示のハードルの低さ**がある。AIへの傾聴に慣れるほど、人間関係での自己開示が苦手になり、リアルなつながりがさらに薄れるという二次的な孤立を生む可能性がある。

#### 孤独が生まれやすい主な場面

| 場面 | 具体的な状況 | 依存しやすさ |
|---|---|---|
| **日常の空き時間** | 帰宅後・休日に話し相手がいない、部屋で一人スマホを見続ける | ◎ 毎日発生する |
| 職場・学校 | テレワーク普及で雑談が消えた、リアルな相談相手がいない | ○ 環境依存だが広範 |
| ライフイベント後 | 引っ越し・転職・離別など人間関係がリセットされるタイミング | ○ 需要が急増する局面 |
| SNS利用中 | 投稿しても反応がない、他者の充実した生活と比較してしまう | ◎ 利用するほど悪化しやすい |
| **AI相談** | 恋愛・人間関係の悩みをAIに話すことが習慣化し、人への開示が減る | ◎ ハードルが低く反復しやすい |

→ **孤独は「繰り返し感じる慢性的な状態」** であるため、毎日使われるサービスへの依存が形成されやすい。「いつでも話を聞いてくれる」「自分を否定しない」存在への需要が高く、AIとの関係性に依存するサービス設計の直接的な根拠となる。さらに、AIへの依存が深まるほど人間関係での摩擦を避けるようになり、孤独と依存が互いを強化するループが生まれる。
