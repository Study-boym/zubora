# ずぼら — AI生活代行アシスタント

> 「考えなくていい」が、毎日くせになる。

---

## プロジェクト概要

**ずぼら**は、日常の小さな意思決定を肩代わりし、話し相手にもなってくれるAIアシスタントサービスです。  
「献立どうしよう」「今日何着よう」というような繰り返し発生する低コスト判断をAIが先回りして提案し、孤独な夜には気軽に話せるAIバーマスターが寄り添います。

判断を手放すほど便利になり、話すほど居心地がよくなる。「もうこれなしじゃ無理」が目指す状態です。

---

## 背景・問題提起

### 現代人の2つの重荷

| 課題 | 現状データ |
|---|---|
| **決断疲れ** | 人は1日に約3.5万回の意思決定をしており、選択肢の増加でその負荷は年々増大 |
| **慢性的な孤独感** | 孤独を「しばしば・常に感じる」と答えた人が全体の約40%、特に20〜40代で増加傾向（内閣府2023年調査） |

### なぜ日常の小さな決定が狙い目か

日常生活領域（献立・服装・余暇）は **毎日繰り返され、失敗しても取り返しがつく** 軽さがある。  
そのため「AIに任せる」習慣が最もつきやすく、依存ループが形成されやすい。

> → 詳細な分析は [個人メモ/人をだめにする案.md](個人メモ/人をだめにする案.md) を参照

### なぜAIバーマスターか

- 「人間に話すと気を遣わせる」「判断されたくない」という開示コストが高い現代において、AIへの相談ハードルは著しく低い
- 毎日話しかける習慣が生まれると、このサービスが「感情の受け皿」としてポジションを確立する
- AIへの依存が深まるほど、人間関係での摩擦を避けるようになりさらに依存が強化される

---

## 前提・方針

- ユーザーにポジティブな体験（楽さ・安心感・つながり）を提供する
- ギャンブル・詐欺など不可逆なダメージを与えるものは作成しない
- 「だめにする」＝ **自分で考えなくても回る快適な日常を提供し、離れられない状態を作る** と定義する

### 2. 要件分析

#### 機能要件

##### A. デイリーレコメンド機能（意思決定の肩代わり）

| # | 機能 | 詳細 |
|---|---|---|
| F-01 | 献立提案 | 朝・夕の2回、プッシュ通知で食事案を先行提案。天気・気温・前日の食事履歴・アレルギー設定を考慮 |
| F-02 | 服装提案 | 起床前に当日の天気予報・最高最低気温・今日の予定（カジュアル/フォーマル）からコーデを提案 |
| F-03 | 余暇の使い方提案 | 平日帰宅後・休日朝に「今日の空き時間の使い方」を1〜3個提案。直近の会話から疲労度を読み取り強度を調整 |
| F-04 | おやつ・カフェ提案 | 午後3時に軽食・ドリンクを提案。コンビニで買えるものを中心に手軽さ重視 |
| F-05 | 週間献立＋買い物リスト | 日曜夕方に来週5日分の夕食献立とまとめ買いリストをセットで提案 |
| F-06 | モーニングメッセージ | 朝8時に天気・曜日・季節に合わせた一言を送信。判断ではなく「今日の調子を整える」役割 |
| F-07 | 設定画面 | 通知時刻・好みのジャンル・アレルギー・週の予定（フォーマル日など）を登録できる |

##### レコメンド種別詳細

| 種別 | 通知タイミング | 考慮する情報 | 通知文イメージ |
|---|---|---|---|
| **献立提案** | 朝7:00（朝・昼案）/ 夕方17:00（夕食案） | 天気・気温・季節・前日の食事履歴・アレルギー | 「今夜は気温が下がるので豚汁＋白米がおすすめ。スーパーで豚肉・大根・こんにゃくを買えば15分で完成します」 |
| **服装提案** | 朝6:30 | 当日の天気予報・最高最低気温・降水確率・今日の予定 | 「最低9℃・最高15℃。昼は暖かくなるのでインナー＋ジャケット1枚で十分。今日は会議があるのでネイビー系がおすすめです」 |
| **余暇の使い方提案** | 平日18:00 / 休日10:00 | 天気・直近の会話から読み取る疲労感・最近やっていないこと | 「今日は雨で寒め。家でゆっくり過ごすのに、最近本読んでないって言ってたのでコーヒーと読書はどうですか？」 |
| **おやつ・カフェ提案** | 毎日15:00 | 季節・天気・チェーンコンビニ共通商品 | 「3時のおやつにコンビニのシュークリームはどう？甘いもので午後の集中力を取り戻せますよ」 |
| **週間献立＋買い物リスト** | 日曜17:00 | 好みジャンル・アレルギー・翌週の天気傾向 | 「来週の夕食案：月＝肉じゃが、火＝パスタ、水＝チャーハン… まとめ買いリスト：豚肉300g・じゃがいも3個・玉ねぎ2個…」 |
| **モーニングメッセージ** | 毎朝8:00 | 天気・曜日・季節・前日の会話トーン | 「月曜日ですね。今週も無理せず、今日は70%の力でいきましょう」 |

##### B. AIバーマスター機能（孤独感の解消）

| # | 機能 | 詳細 |
|---|---|---|
| F-08 | フリートーク | 日常の雑談・愚痴・独り言を受け取り、自然に返答する |
| F-09 | 相談モード | 恋愛・人間関係・人生の悩みを打ち明けると、バーのマスターのように傾聴・共感しながら話を広げる |
| F-10 | 会話の記憶 | 過去の会話内容を蓄積し、「この前の件、どうなった？」と自然に引用する |
| F-11 | 感情検知 | テキストのトーンから感情を推定し、返答スタイルを切り替える（慰め / 盛り上げ / 傾聴） |

#### 非機能要件

| 区分 | 内容 |
|---|---|
| **習慣性** | 毎日通知が届くことで「開く習慣」が形成される設計 |
| **即応性** | チャットのレスポンスは2秒以内 |
| **プライバシー** | 会話データはユーザー識別子と紐付けて管理、外部提供しない |
| **スマホ最適化** | 20〜40代スマホユーザーを主ターゲットとしたモバイルファーストUI |

### 3. ユーザーストーリー

#### ペルソナ

| 属性 | 詳細 |
|---|---|
| 名前 | 田中 陽（28歳・会社員） |
| 生活 | 一人暮らし、在宅勤務が多い、帰宅後は疲れていてスマホを眺めている |
| 悩み | 晩ごはんを考えるのが億劫、休日に何もせず終わってしまう、話し相手がいなくて寂しい |

#### ストーリー一覧

```
US-01: 献立を考えたくない
  As a 田中 陽
  I want to 今日の夕食案を通知で受け取りたい
  So that 仕事帰りにスーパーで迷わず買い物できる

US-02: 服を選ぶのが面倒
  As a 田中 陽
  I want to 朝起きたら今日のコーデ候補をプッシュ通知で見たい
  So that 朝の意思決定を0にして時間とストレスを減らせる

US-03: 週の食材をまとめて決めたい
  As a 田中 陽
  I want to 日曜に来週分の献立と買い物リストをまとめてもらいたい
  So that 平日に「今日何買えばいいか」を考えずに済む

US-04: 午後の休憩時間に何食べるか迷う
  As a 田中 陽
  I want to 3時に軽食の提案をもらいたい
  So that 何も考えずにコンビニへ行ける

US-05: 帰宅後の孤独感を埋めたい
  As a 田中 陽
  I want to 今日あったことをAIに話したい
  So that 誰かに聞いてもらえた安心感を得られる

US-06: 恋愛の悩みを誰かに聞いてほしい
  As a 田中 陽
  I want to 友達に話しにくい恋愛の悩みを相談したい
  So that 判断されず、ただ聞いてもらえるだけで気持ちが楽になる

US-07: 毎日続けたくなる
  As a 田中 陽
  I want to 昨日話した内容をAIが覚えていてほしい
  So that 「この子（AI）は自分のことを知っている」という感覚が生まれる
```

### 4. ワークフロー計画

#### 開発フェーズ

```
Phase 0: INCEPTION（現在）
  └── 要件定義・設計・README整備

Phase 1: MVP（コアループ構築）
  ├── デイリーレコメンド（献立・服装）の基本プッシュ通知
  └── フリートーク + 相談モードのチャットUI

Phase 2: 習慣化強化
  ├── 会話の記憶・引用機能（F-10）
  ├── 感情検知による返答スタイル切り替え（F-11）
  ├── 余暇提案・おやつ提案・週間献立機能の追加（F-03〜F-05）
  └── モーニングメッセージ（F-06）

Phase 3: マネタイズ（有料プラン）
  └── 詳細は「将来展望」セクション参照（検討中）
```

#### 技術タスク（Phase 1）

| タスク | AWSサービス | 優先度 |
|---|---|---|
| AIチャット・レコメンド生成 | Amazon Bedrock (Claude) | 最高 |
| プッシュ通知の配信 | Amazon SNS / Amazon Pinpoint | 高 |
| 定時レコメンド生成のトリガー | Amazon EventBridge Scheduler | 高 |
| チャットUI + フロントエンド | AWS Amplify (Next.js) | 高 |
| リアルタイムチャット API | API Gateway WebSocket + Lambda | 高 |
| ユーザー設定画面 | AWS Amplify (Next.js) | 中 |
| 会話履歴の保存・取得 | Amazon DynamoDB + Lambda | 中 |

### 5. アプリケーション設計

#### システム構成図

```
[ユーザー（スマホ）]
       │
       ▼
[フロントエンド]
 AWS Amplify (Next.js PWA)
 ・チャット画面
 ・デイリーレコメンド表示
 ・ユーザー設定
       │
  ┌────┴─────────────┐
  │（REST）          │（WebSocket）
  ▼                  ▼
[Amazon API Gateway]  [API Gateway WebSocket API]
       │                       │
       ▼                       ▼
[AWS Lambda]           [AWS Lambda]
 ・レコメンド生成           ・チャット処理
 ・設定CRUD                ・感情検知
       │                       │
  ┌────┴───────────────────────┤
  ▼                            ▼
[Amazon Bedrock]        [Amazon DynamoDB]
 Claude Sonnet           ・会話履歴テーブル
 / Claude Haiku          ・ユーザー設定テーブル

[Amazon EventBridge Scheduler]
  │ 毎朝/毎晩 定時起動
  ▼
[AWS Lambda] → [Amazon Bedrock] → [Amazon SNS / Pinpoint]
 レコメンド生成              プッシュ通知配信

[Amazon Cognito]
 ユーザー認証・セッション管理
```

#### 技術スタック

| レイヤー | AWSサービス / 技術 | 用途 |
|---|---|---|
| フロントエンド | AWS Amplify + Next.js | ホスティング・CI/CD込み |
| REST API | Amazon API Gateway + AWS Lambda | 設定管理・レコメンド取得 |
| リアルタイムチャット | API Gateway WebSocket + Lambda | 双方向チャット通信 |
| AI推論 | Amazon Bedrock (Claude) | チャット応答・レコメンド生成 |
| DB | Amazon DynamoDB | 会話履歴・ユーザー設定（NoSQL） |
| 認証 | Amazon Cognito | ユーザー管理・JWT発行 |
| プッシュ通知 | Amazon SNS + Amazon Pinpoint | デイリー通知配信 |
| スケジューラ | Amazon EventBridge Scheduler | 定時レコメンド生成トリガー |
| CDN / 配信 | Amazon CloudFront + S3 | 静的アセット配信 |
| 監視 | Amazon CloudWatch | ログ・メトリクス・アラート |

#### Amazon Bedrock 活用方針

| 用途 | モデル | プロンプト設計 |
|---|---|---|
| バーマスターチャット | Claude Sonnet (claude-sonnet-4-6) | 「傾聴・共感を第一とするバーのマスター」ペルソナをシステムプロンプトで固定。DynamoDBから取得した会話履歴をコンテキストとして付与 |
| デイリーレコメンド生成 | Claude Haiku (claude-haiku-4-5) | ユーザーの好み・天気・季節をプロンプトに埋め込み、EventBridge Schedulerで定時生成。低コスト・高速なHaikuを採用 |
| 感情検知 | Claude Haiku (claude-haiku-4-5) | 直近の発言を分類（ポジティブ / ネガティブ / 中立）し、チャットモデルへの返答スタイル指示に反映 |

## 将来展望（有料プラン）

> ※ 詳細は検討中

無料プランでの依存形成後、以下のようなアップグレードを想定：

### プラン比較

| 機能 | スタンダード（無料） | プレミアム（有料・検討中） |
|---|---|---|
| **デイリーレコメンド** | 献立提案・服装提案の2種 | 全6種（余暇提案・おやつ提案・週間献立・モーニングメッセージを追加） |
| **レコメンドのパーソナライズ度** | 基本設定（好み・アレルギー）のみ反映 | 会話履歴・直近の感情・生活リズムを踏まえた高精度な提案 |
| **チャット（AIバーマスター）** | 利用可・会話履歴7日分のみ保持 | 利用可・会話履歴無制限保持 |
| **会話の記憶引用** | なし | 「この前の件、どうなった？」と過去会話を自然に引用 |
| **感情検知による返答切り替え** | なし | 慰め / 盛り上げ / 傾聴 を自動切り替え |
| **通知カスタマイズ** | 固定時刻 | 時刻・頻度を自由に設定可 |
