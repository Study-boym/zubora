# ずぼら — AI生活代行アシスタント

> 「考えなくていい」が、毎日くせになる。

---

## 一言で

ずぼらは、毎日の小さな意思決定に疲れ、人と話すハードルが高いと感じている人のための、AI生活代行アシスタントです。

献立・服装・余暇の判断をAIが先回りして通知し、愚痴や悩みはバーのマスターのように聞いてくれます。使えば使うほど「自分のことをわかってくれる」存在になる。それが、ずぼらが人を「だめにする」仕組みです。

## なぜ作るのか

現代人の日常には、2つの慢性的な重荷があります。

**意思決定疲れ**: 人は1日に約3.5万回の意思決定をしています。仕事で認知リソースを使い切った後に、「夕食は何にする？」「今夜どう過ごす？」という判断が重なります。選択肢が多いほど選べなくなり、結局スマホをぼーっと見て夜が終わる。

**日常的な孤独**: 内閣府の調査（2023年）では、孤独を「しばしば・常に感じる」と答えた人が全体の約40%にのぼり、特に20〜40代で増加傾向にあります。SNSで「つながっている」ように見えても、人に話しかけるハードルの高さや、気軽に打ち明けられる相手の不在という問題は解消されていません。

ずぼらは、この2つを同時に解決します。判断を代行し、話し相手になる。使えば使うほど快適になる、「人をだめにする」AIパートナーサービスです。

## 対象ユーザー

- 仕事後に食事・服装・夜の過ごし方を考える気力が残っていない人
- 人に話しかけるハードルが高く、愚痴や悩みを気軽に打ち明けられる相手がいない人
- 誰かに「聞いてもらえた」という安心感を日常的に求めている人

## MVP 体験

### 1. デイリーレコメンド

献立・服装・余暇など、毎日繰り返す低コストな判断をAIが先回りしてプッシュ通知で届けます。

- 比較リストではなく「今日はこれです」と一択を提示
- 天気・前日の食事履歴・アレルギー設定・今日の予定をもとに個別化
- 必要な準備（買い物リスト・コンビニ商品情報など）もあわせて提示

### 2. AIバーマスター（チャット）

帰宅後や隙間時間の雑談・愚痴・悩みを、バーのマスターのように受け取ります。

- 判断せず、ただ聞いてくれる安心感
- 過去の会話を記憶し、「この前の件、どうなった？」と自然に引用
- テキストのトーンから感情を読み取り、慰め・盛り上げ・傾聴を自動切替

## ずぼらの一日

**06:30 — 服装通知が届く**

> 「おはようございます。今日は最低9℃・最高15℃、午後から雨の予報です。
>
> インナー＋ジャケット1枚で。今日は会議があるのでネイビー系がおすすめです。折りたたみ傘を忘れずに。」

天気・気温・今日の予定をもとに、服装方針を一択で提示します。ユーザーは迷いません。

**17:00 — 夕食案が届く**

> 「帰りに豚汁の材料を買っておきましょう。豚肉・大根・こんにゃく・味噌で15分で完成します。
>
> 18時半にスーパーが割引になるので、帰宅ルートに合わせて寄り道を。」

気温・季節・前日の食事履歴から夕食案を一択で提示し、買い物リストまで添えます。

**19:45 — 帰宅後のチャット（AIバーマスター）**

> 「おかえりなさい。今日はどうでしたか？」

> （ユーザー）「会議が長引いて疲れた」

> 「それは消耗しますね。今夜はとにかくゆっくりしましょう。
>
> 豚汁を温めながら、何か見ますか？最近ドラマ見てないって言ってたので、一緒に探しましょうか。」

雑談を受け取り、感情トーンを読んで返答スタイルを切り替えます。過去の会話も自然に引用します。

## AWSアーキテクチャ

- **Amazon Bedrock (Claude Sonnet)**: バーマスターチャット応答・レコメンド生成
- **AWS Lambda**: チャット処理・レコメンド生成オーケストレーション
- **Amazon API Gateway (WebSocket)**: リアルタイムチャットAPI
- **Amazon API Gateway (REST)**: レコメンド取得・ユーザー設定CRUD
- **Amazon DynamoDB**: 会話履歴・ユーザー設定管理
- **Amazon EventBridge Scheduler**: 定時レコメンド生成トリガー（朝・夕・週次）
- **Amazon SNS / Amazon Pinpoint**: デイリー通知の配信
- **AWS Amplify + Amazon CloudFront / S3**: フロントエンドホスティング・静的アセット配信
- **Amazon Cognito**: ユーザー認証・セッション管理
- **Amazon CloudWatch**: ログ・メトリクス・アラート

## 審査基準への対応

- **課題の明確さ**: 意思決定疲れ（1日3.5万回の判断）と孤独・コミュニケーション障壁（40%が慢性的に孤独を感じる）という2軸の課題を定量データで定義します。
- **テーマ適合性**: 判断を任せるほどAIへの依存が深まり「だめになる」体験を、デイリーレコメンドとチャットの2軸でプロダクトのコアメカニクスとして表現します。
- **実現可能性**: MVPをデイリーレコメンドとフリートークチャットに絞り、AWSのマネージドサービスで構成します。
- **トレーサビリティ**: 要件・ストーリー・アーキテクチャ・ユニット分解を `docs/inception/` 配下で追跡します。
- **安全性**: 医療・法律・金融・重大な人間関係などの高リスク判断は対象外とし、会話データはユーザー識別子と紐付けて管理・外部提供しません。

## 今後の展開

書類審査通過後のConstructionフェーズで、以下をMVPプロトタイプとして実装します。

- **Phase 1 — コアループ**: デイリーレコメンド（献立・服装）のプッシュ通知 ＋ フリートーク・相談モードのチャットUI
- **Phase 2 — 習慣化強化**: 会話の記憶引用・感情検知による返答切替・余暇/おやつ/週間献立提案の追加
- **Phase 3 — マネタイズ**: 会話履歴の無制限保持・高精度パーソナライズなどの有料プラン実装
